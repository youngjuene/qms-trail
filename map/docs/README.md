# Documentation

Project documentation organized by purpose.

## Structure

### `/design` - Architecture & Specifications
System design, architecture decisions, and technical specifications.

- **PHOTO_ARCHIVE_DESIGN.md** - Complete system design and architecture

### `/implementation` - Implementation Logs
All implementation summaries, status updates, and change logs.

- **IMPLEMENTATION_STATUS.md** - Current implementation status
- **IMPLEMENTATION_SUMMARY.md** - Latest implementation summary
- **DATABASE_CLEANUP_SUMMARY.md** - Database cleanup notes
- **FOV_METADATA_FIXES.md** - Field of view metadata fixes
- **IMPLEMENTATION_COMPLETE.md** - Completion milestone
- **IMPLEMENTATION_TEST_REPORT.md** - Test results
- **SPACEBAR_FOV_ADJUSTMENT.md** - Spacebar FOV adjustment feature
- **STATS_LAYOUT_IMPROVEMENT.md** - Stats layout changes
- **TROUBLESHOOTING_FIXES_2025_10_30.md** - Troubleshooting fixes
- **UPLOAD_MODAL_SIMPLIFICATION.md** - Upload modal updates

### `/guides` - Integration & Workflow Guides
User guides, integration documentation, and workflow descriptions.

- **INTEGRATION_GUIDE.md** - Frontend integration guide
- **UPLOAD_WORKFLOW.md** - Photo upload workflow documentation

## Document Lifecycle

- New architecture/design docs → `/design`
- Implementation logs and status → `/implementation`
- User/integration guides → `/guides`
